# Data

All data in this project is synthetic, generated by `phishnet.data.generate_dataset`.
There is no bundled or downloaded corpus, so there is no license or attribution
to track here — the generator itself is the "dataset" and lives in
[`src/phishnet/data.py`](../src/phishnet/data.py).

Phishing emails are built from 10 templates covering common tactics (fake
prizes, account-blocked threats, payment pressure, tax scares, tech-support
scams, credential harvesting), each filled in with randomized brand names,
links, deadlines, and reference numbers. Normal emails are built the same way
from 10 templates covering everyday academic/work correspondence. The class
ratio defaults to 35% phishing, 65% normal, to roughly mirror the minority-class
imbalance seen in real phishing datasets.

This is enough to demonstrate the modeling pipeline (train/val/test split,
vocabulary built only from the training split, baseline vs. attention-model
comparison) end to end, but it is not a substitute for a real, labeled corpus.
See the README's Limitations section for what that means for the reported
numbers.
